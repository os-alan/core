In addition to properties on the grid proper, properties on individual columns can also be set declaratively through grid state, keyed by column name.

Although column (or row, or cell) properties can be set via grid.setState, this method is not appropriate when your app’s columns (or rows, or cells) are not static. When you don’t know about the columns in advance, you can set column properties by other means, e.g., programmatically, or by loading and setting column properties objects, etc.
State for columns, rows, and cells may be set by dynamic properties bearing those names, which have the following shapes.

For example, to set properties on column prevclose:

{ columns: { prevclose: { key: 'value' } } }

To set properties on row the fourth data row:

{ rows: { data: { 3: { key: 'value' } } } }

To set properties on the cell at the intersection of the fourth data row and column prevclose:

{ cells: { data: { 3: { prevclose: { key: 'value' } } } } }
Properties cascade:
  • A property set on a specific cell would override the same property set on the cell’s column.
  • A property set on a specific column would override the same property set on the grid.
  • A property set on the grid would override the same property set on an active theme or the properties default value.
The exact ordering of the cascade, which can also include row properties, can itself be specified by the propClassLayers grid property.
