Localizers tab

For a grid cell to use a localizer to format its value, three coding steps are required:
  1. Define the localizer
    Code the localizer in the Localizer tab's editor.
  2. Register the localizer
    Use the Localizer tab’s grid.localization.add(…) button.
  3. Reference the localizer
    Cells reference the localizer by name in the format property. Use the State tab's editor to set the format property declaratively, either for the whole grid or for a specific column or cell.
The activity on the next page uses the workbench tabs to perform all three steps. On the page after that, we’ll repeat it programmatically in code.

Define & register a new localizer
  1. Switch to the Localizers tab.
  2. Select (New) from the Load localizer drop-down.
  3. Rename it with a unique name by editing the name field.
  4. Add additional local variables by declaring them at the top, as needed.
  5. Code your localizer referring to examples and the localizer interface documentation.
  6. Be sure to assign your localizer to module.exports.
For example, here’s a file containing a localizer to show a “degrees” unit: 
module.exports = {
    name: 'deg',
    format: function(s) { return s + '°' },
    parse: function(s) { return parseInt(s); }
}
Now register your new localizer with the grid:
  1. Click the grid.localization.add(…) button.
The grid.localization.add(…) button can also be used to update a localizer already known to the grid:
  1. Make a minor edit to your localizer code.
  2. Click the grid.localization.add(…) button again.
